# Tunnelhunt MCP Server

Официальный сервер Model Context Protocol (MCP) для платформы **Tunnelhunt**. 

Этот сервер позволяет ИИ-агентам (таким как Claude Desktop, Cursor, Cline) нативно взаимодействовать с вашими туннелями. Агенты могут автоматически пробрасывать локальные порты в интернет, проверять статусы запросов и закрывать туннели без ручного ввода команд в терминал. Это идеальное решение для **Vibe Coding**, когда агенту нужен публичный URL для тестирования написанного им веб-приложения или вебхуков.

## Доступные инструменты (Tools)

MCP-сервер предоставляет ИИ-агентам следующие инструменты:

1. **`start_tunnel`**
   - **Описание:** Запускает SSH-туннель к локальному порту.
   - **Аргументы:** 
     - `port` (number, required): Локальный порт для проброса (например, 3000).
     - `subdomain` (string, optional): Желаемый субдомен. Если не указан, будет сгенерирован случайный.
   - **Возвращает:** Публичный HTTPS URL туннеля.

2. **`stop_tunnel`**
   - **Описание:** Закрывает ранее открытый туннель.
   - **Аргументы:**
     - `subdomain` (string, required): Субдомен туннеля, который нужно закрыть.

3. **`get_tunnel_logs`**
   - **Описание:** Возвращает недавние логи HTTP-трафика для туннеля.
   - **Аргументы:**
     - `subdomain` (string, required): Субдомен туннеля.
   - **Применение:** Позволяет ИИ-агенту убедиться, что вебхуки от сторонних сервисов (например, Telegram, Stripe) успешно дошли до локального сервера, и посмотреть их статусы ответа.

## Использование через NPM

Пакет опубликован в NPM как `@tunnelhunt/mcp`. Это значит, что вам не нужно клонировать репозиторий и собирать его вручную — ИИ-клиенты могут запускать его напрямую через `npx`.

### Настройка в популярных ИИ-клиентах

### 1. Claude Desktop
Откройте файл конфигурации Claude Desktop.
- **macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

Добавьте следующую конфигурацию в массив `mcpServers`:

```json
{
  "mcpServers": {
    "tunnelhunt": {
      "command": "npx",
      "args": [
        "-y",
        "@tunnelhunt/mcp"
      ]
    }
  }
}
```
После этого перезапустите Claude Desktop.

### 2. Cursor
1. Откройте **Settings** (Настройки) в Cursor.
2. Перейдите в раздел **Features -> MCP**.
3. Нажмите **+ Add New MCP Server**.
4. Выберите тип: `command`.
5. Введите имя: `tunnelhunt`.
6. Введите команду: `npx -y @tunnelhunt/mcp`.
7. Сохраните и убедитесь, что сервер загорелся зеленым (Connected).

### 3. Cline (VS Code Extension)
1. В панели Cline нажмите на иконку **MCP Servers** (значок сервера).
2. Нажмите **Edit MCP Settings**.
3. Добавьте сервер в `cline_mcp_settings.json`:

```json
{
  "mcpServers": {
    "tunnelhunt": {
      "command": "npx",
      "args": [
        "-y",
        "@tunnelhunt/mcp"
      ]
    }
  }
}
```

## Переменные окружения (.env)
Если необходимо переопределить стандартный домен и порт сервера, вы можете передавать их через блок `env` в настройках MCP-сервера в вашем клиенте (например, в Claude или Cline):

```json
{
  "mcpServers": {
    "tunnelhunt": {
      "command": "npx",
      "args": [
        "-y",
        "@tunnelhunt/mcp"
      ],
      "env": {
        "TUNNEL_DOMAIN": "tunnelhunt.ru",
        "TUNNEL_PORT": "2222"
      }
    }
  }
}
```

## Локальная разработка

Если вы хотите внести изменения в код сервера:
```bash
git clone https://github.com/tunnelhunt/mcp.git
cd mcp
npm install
npm run build
```
Для использования локальной версии замените команду `npx` в настройках вашего клиента на вызов `node` с абсолютным путём до скомпилированного файла `build/index.js`.

## Как это работает?
MCP-сервер выступает оберткой над вашей локальной утилитой `ssh` (которая уже встроена в macOS/Linux и Windows 10+). Когда агент вызывает `start_tunnel`, сервер под капотом запускает процесс `ssh -R ...`, парсит его вывод (чтобы узнать выделенный субдомен) и держит процесс открытым. Он также перехватывает `stdout`, чтобы собирать логи входящих HTTP-запросов и отдавать их через `get_tunnel_logs`.
